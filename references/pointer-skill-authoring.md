# Pointer Skill Authoring

SOS pointer skills are lightweight active skills generated by `sos apply --apply`.
They keep the full source skill in the SOS vault and point the agent to the pack
manifest that names the vault copy.

Current generated active skills:

- `sos-haruhi`: companion entry that references the runtime registry.
- `sos-apify`: pointer for the Apify pack.
- `sos-obsidian`: pointer for the Obsidian pack.
- `sos-game-design`: pointer for the Game Design pack.

The implemented command flow is:

1. `sos scan --root <skills> [--codex-config <config.toml>]`
2. `sos propose --root <skills>`
3. `sos plan --root <skills> --runtime-root <.sos> --codex-config <config.toml> --out <plan.toml>`
4. `sos apply --plan <plan.toml>` for dry-run summary.
5. `sos apply --plan <plan.toml> --apply` for writes.
6. `sos status --runtime-root <.sos>`
7. `sos pack activate <pack> --runtime-root <.sos> [--sync clean-auto]`
8. `sos pack sync <pack> --runtime-root <.sos> [--apply]`

Authoring rules:

- Use `sos-` prefixed skill names only.
- Keep pointer text short; do not embed copied source skill bodies.
- Include the manifest path and vault root so the agent can locate the pack.
- Treat writes as explicit: `sos apply --apply` executes the audited apply plan, and
  `sos pack sync --apply` writes eligible sync changes. `sos pack activate <pack>
  --sync clean-auto` can also write the vault copy and manifest when clean source
  drift is eligible, but it never changes Codex config or deletes source skills.
- Do not document pointer behavior that is not represented in manifests or tests.
